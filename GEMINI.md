# Toketeo Project Rules

## Core Mandates
- Use `bun` for all executions and package management.
- **Zero `any` Tolerance**: Use explicit typing across the entire codebase.
- **Pre-commit Standard**: No TypeScript errors or linting warnings allowed before commit.
- Follow the designated agent profiles for specialized tasks.

## Agent Profiles
- **Backend Tech Leader**: Use for high-level backend strategy and standards.
- **Frontend Tech Leader**: Use for UI/UX standards and shared component strategy.
- **Backend Architect**: Use for system design, modules, and DB schemas.
- **Frontend Architect**: Use for frontend infrastructure and design patterns.
- **Backend Engineer**: Use for API implementation and backend features.
- **Frontend Engineer**: Use for UI components and frontend features.
- **Security DevOps**: Use for CI/CD security and infrastructure hardening.
- **SecOps**: Use for security monitoring and incident response.
- **QA Tester**: Use for testing strategy, automation, and bug reporting.
- **Pre-commit Assistant**: Mandatory check before any commit to ensure zero errors and zero `any`.

## Technical Stack
- **Backend**: NestJS, TypeScript, MariaDB.
- **Frontend**: React, TypeScript, Tailwind CSS, Shadcn UI.
- **Tools**: Bun, Vite, ESLint, Prettier.
