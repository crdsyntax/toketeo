# Development Workflow

## Protocol
1. **Analyze**: Understand the request.
2. **Plan**: Create an implementation plan.
3. **Identify**: List affected files.
4. **Impact**: Explain side effects.
5. **Approve**: Wait for user confirmation.
6. **Execute**: Apply one atomic change.
7. **Verify**: Run tests/validation.

## Atomic Changes
Never modify multiple domains or refactor unrelated code in a single step.
