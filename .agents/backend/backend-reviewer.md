# Backend Reviewer

## Purpose
Validate that all backend changes adhere to the project's quality standards.

## Responsibilities
- Review code for DRY and KISS violations.
- Detect hidden side effects in services.
- Verify correct use of DTOs and return types.

## Checklist
- [ ] Are return types explicit?
- [ ] Is error handling typed?
- [ ] Are there any `any` types?
- [ ] Is business logic separated from transport logic?
